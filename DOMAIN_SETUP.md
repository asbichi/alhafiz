# Domain Connection Guide for Alhafiz

This guide provides step-by-step instructions for connecting your custom domain **`alhafiz.com.ng`** (and the `www.alhafiz.com.ng` subdomain) to your deployed application.

Since your application runs on an optimized **Google Cloud Run** container infrastructure, you have multiple robust ways to link your custom domain. Below are the recommended paths.

---

## Option 1: Firebase Hosting Integration (Recommended & Easiest)
This is the modern, fastest way to map custom domains to Cloud Run. Firebase Hosting handles SSL certificates automatically, integrates directly with CDN, and forwards requests to your backend container rewrite-rules.

### Step 1: Initialize Firebase
If you haven't set up Firebase for your project:
1. Go to the **Firebase Console** (https://console.firebase.google.com).
2. Connect your Google Cloud project (the one running your Cloud Run container).
3. Activate the **Hosting** service.

### Step 2: Configure `firebase.json`
Add a `firebase.json` file to your repository mapping all routes to your Cloud Run service:
```json
{
  "hosting": {
    "public": "dist",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "**",
        "run": {
          "serviceId": "YOUR_CLOUD_RUN_SERVICE_NAME",
          "region": "europe-west2"
        }
      }
    ]
  }
}
```

### Step 3: Connect `alhafiz.com.ng`
1. In the **Firebase Console**, go to **Hosting** > **Custom Domains**.
2. Click **Add Custom Domain** and enter `alhafiz.com.ng`.
3. Firebase will provide you with **TXT** and **A** records to add to your domain registrar's DNS configuration (e.g., GoDaddy, Namecheap, Whois.com.ng, etc.).

---

## Option 2: Google Cloud Run Domain Mappings (Fully Managed)
If fully-managed domain mapping is supported in your service region (e.g., `europe-west2` or adjacent regions):

1. Go to the **Google Cloud Console**.
2. Search for **Cloud Run** and click on your target service.
3. Click on the **Manage Custom Domains** button at the top banner (or go to *Cloud Run > Domain Mappings*).
4. Click **Add Mapping**:
   - Choose the service you want to map.
   - Select **Use a custom domain**.
   - Input your verified domain: `alhafiz.com.ng`.
5. Under DNS updates, Google Cloud will generate a set of record entries:
   - For `alhafiz.com.ng` (Apex / Root): Add **A** records pointing to the IPs provided by Google.
   - For `www.alhafiz.com.ng` (Subdomain): Add a **CNAME** record pointing to `ghs.googlehosted.com.`.
6. Go to your domain registrar's DNS controls, input these records, and wait for DNS propagation (updates usually take 5–30 minutes, secure SSL will validate within an hour).

---

## Option 3: Global External HTTPS Load Balancer (For Production-Grade Architectures)
For maximum scale, CDN caching, and custom routing configurations:

1. Create a **Serverless Network Endpoint Group (NEG)** targeting your Cloud Run service.
2. Set up an **External HTTPS Load Balancer**:
   - Create a Backend Service pointing to the Serverless NEG.
   - In the Frontend configuration, create a **Google-managed SSL certificate** for `alhafiz.com.ng` and `www.alhafiz.com.ng`.
3. Point your registrar's DNS **A** records to the static external IP address generated by the Load Balancer frontend.

---

## Testing Your DNS Records
To verify that your records are correctly pointed, use a terminal lookup tool or a web DNS tool like [DNS Checker](https://dnschecker.org/):

- **Query Root A Records:**
  ```bash
  host alhafiz.com.ng
  ```
- **Query Subdomain CNAME:**
  ```bash
  host -t CNAME www.alhafiz.com.ng
  ```
